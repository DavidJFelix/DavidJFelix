## Decisions

- Don't build a new tool for transitive freshness. Renovate's `lockFileMaintenance` is the canonical answer.
- **Single repo-level Renovate config** at `.github/renovate.json` covers all apps, exercises, and Cargo workspaces — no more per-app `renovate.json` files.
- **Renovate is the sole dependency automation tool.** Dependabot version updates were never configured; Dependabot security updates get turned off in the repo Settings UI (human task — filed as an issue).
- **Run Renovate via GitHub Actions, not the Mend cloud app.** Matches the pattern David wants to replicate across closed-source repos. Auth via `RENOVATE_TOKEN` (GitHub App recommended over PAT).
- **Batched PRs**: one grouped PR per ecosystem per week for patch + minor (`npm`, `cargo`, `github-actions`, `mise`). Majors stay as their own PRs.
- **Vulnerability handling**: Renovate's `vulnerabilityAlerts` replaces Dependabot security PRs.
- Schedule: weekly, Monday before 5am UTC — aligns with the existing freshness cron.
