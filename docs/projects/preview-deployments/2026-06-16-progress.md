# 2026-06-16 — Preview Deployments Phase 1 (f311x)

## Session summary

Built the f311x slice of per-PR preview deployments end to end: deploy an isolated Alchemy stage per
PR, smoke it, screenshot-diff it against committed baselines, comment the URL, and tear it down on
close. This is the keystone that unblocks dependency-freshness auto-merge (Phase 5).

## Work completed

- **Per-PR preview deploy** — `apps/f311x/bin/deploy-preview.ts` deploys stage `pr-<N>` (non-prod ⇒
  no custom domain, only `*.workers.dev`). Alchemy names the worker
  `f311x-website-<stage>-<random16>` with a state-persisted suffix, so the script resolves the live
  URL from the Cloudflare API after deploy (match `f311x-website-pr-<N>-*` against
  `/workers/scripts`, join with the account `/workers/subdomain`) and emits it to `$GITHUB_OUTPUT`.
  Hang-after-success tolerance mirrors `bin/deploy-prod.ts`.
- **Teardown** — `apps/f311x/bin/destroy-preview.ts` (`alchemy destroy --stage pr-<N>`), best-effort
  so a missing stage never fails the job.
- **Smoke against the preview** — reuses the existing `bin/smoke-test.ts` / `bin/smoke-checks.ts`
  (hydration + chat-echo) with `SMOKE_URLS=<preview>`.
- **Screenshot regression** — `playwright.config.ts` + `src/routes/index.e2e.test.ts` using
  `toHaveScreenshot`. Config targets `PREVIEW_URL` when set, else boots the built worker via
  `wrangler dev` locally (so the chat backend is live, unlike `vite preview`). Committed baseline:
  `src/routes/index.e2e.test.ts-snapshots/home-chromium-linux.png`.
- **CI** — `.github/workflows/cd-preview-f311x.yml`: deploy → smoke → screenshots → sticky PR
  comment (`bin/comment-preview.ts`) on every PR touching `apps/f311x`; teardown job on `closed`.
  Concurrency-grouped per PR.
- **Plumbing** — `@playwright/test@1.60.0` + `test:e2e*` scripts and a `test:e2e` mise task; vitest
  now excludes `*.e2e.test.ts`; the route generator ignores `*.test.*` (`vite.config.ts`) so the
  spec co-locates in `src/routes/` without becoming an endpoint; `test-results/` +
  `playwright-report/` gitignored. SessionStart hook now installs Playwright chromium (the deferred
  testing-harness follow-up) so e2e runs in web sessions.

## Decisions

- Committed baselines over compare-against-production (deterministic, reviewable).
- Playwright `toHaveScreenshot` over a hosted visual-diff service.
- Preview fires on every f311x PR (no label gate); low PR volume makes the real-deploy cost a
  non-issue.
- Fork PRs (no secrets) are left to fail their preview job; not switching to `pull_request_target`.

## Verification

Local (this Linux container, matching ubuntu CI): `build`, `typecheck`, `lint`, `format`, vitest
(100% logic coverage, e2e excluded), and `mise run test:e2e` all green; both e2e tests pass against
the committed baseline via the local `wrangler dev` boot. Bin scripts exercised via their test seams
(`DEPLOY_PREVIEW_TEST_CMD` / `DESTROY_PREVIEW_TEST_CMD`) — deploy/credential/PR guards behave. The
real deploy path needs the `F311X_CLOUDFLARE_*` secrets and is verified by opening a PR (see Next
steps).
