# 2026-06-11 Progress

## Summary

Project created during a portfolio review session. f311x is broken in
production despite green CI and a successful deploy, which exposed two missing
capabilities: reading why production is broken (observability) and seeing that
a change will be broken before it merges (preview deploys + smoke/screenshot
tests). David rated this the highest concern. The same capability is also the
new bar for dependency auto-merge: no auto-merge for web apps without preview
verification.

## Decisions made

- Preview + smoke + screenshot testing is its own project, not a sub-task of
  dependency-freshness; auto-merge is a downstream beneficiary, not the driver.
- f311x is the proving ground (Phase 1); generalize to the other apps after.
- Production observability is coordinated with the existing Sentry Integration
  project (and Cloudflare Workers logs) rather than duplicated here.

## Next steps

- Diagnose the current f311x breakage (also the top item on the f311x plan).
- Design the Alchemy preview-stage setup for per-PR deploys.

## Blockers or open questions

- Screenshot baseline strategy (preview vs production, or committed baseline).
- Visual diff tooling choice (Playwright screenshots vs dedicated service).
