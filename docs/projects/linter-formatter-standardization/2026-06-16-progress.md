## Summary

Landed the decision-free "mechanical slice" the 2026-06-11 audit carved out: cspell
is now actually enforced repo-wide, the broken root `.prettierignore` is fixed, the
two drifting Biome schemas are bumped, and the Phase 4 docs (`docs/tooling-standard.md`
+ CLAUDE.md) exist. The three scope decisions (Rust depth, legacy JS dirs, root-markdown
enforcement) are untouched and still pending.

## Work completed

- **cspell — one repo-wide gate.** Added cspell as a root tool via mise's npm backend
  (`"npm:cspell" = "10.0.1"` in `.config/mise.toml`, lockfile committed) and a root
  `[tasks.spell]` scoped to the repo's own sources (`apps/`, `docs/`, `bin/`, root
  markdown, `.config/`, `.github/`). New `.github/workflows/ci-spell.yml` runs
  `mise run spell` on every push/PR with no paths filter.
- **Triaged the cspell backlog to green.** First run surfaced 356 issues. Extended
  `.config/cspell.json` `ignorePaths` to cover generated/build trees (`.svelte-kit`,
  `.nuxt`, `.output`, `.astro`, `.vinxi`, `.wrangler`, `coverage`, `*.gen.ts`, …),
  which cleared the bulk. Added ~80 legitimate domain terms to the shared dictionary.
  One real typo was a deliberate quote in a changelog ("Fixed typo (`applicaiton`)") —
  handled with an inline `cspell:ignore` rather than whitelisting the misspelling.
  Verified the gate fails on an injected misspelling and passes after revert.
- **Consolidation.** Removed the redundant per-app `spell` scripts from all 11 apps
  (6 were already broken — no cspell devDep) and the `cspell` + `@cspell/dict-typescript`
  devDeps from the 5 apps that carried them; refreshed those 5 lockfiles (clean diffs —
  only the cspell tree was pruned).
- **`.prettierignore`.** Replaced the non-functional `*` + `!**/*.md` inversion with the
  working `**` + `!*/` + `!*.md`/`!*.mdx` recipe plus explicit generated-tree excludes.
  Verified: a bare `prettier .` flags 0 non-md files, sees all 48 `docs/` markdown files,
  and leaks nothing from `node_modules`/`styled-system`.
- **Biome schema bump.** `apps/davidjfelix.com/biome.json` and `apps/djf.io/biome.json`
  2.4.4 → 2.4.16; `biome check` green on both (no config-key migration needed).
- **Phase 4 docs.** New `docs/tooling-standard.md` (ownership map per ecosystem +
  enforcement wiring + open caveats); CLAUDE.md Tooling section gained cspell-as-gate,
  tsgo, a Rust (aspirational) line, and a pointer to the new doc.

## Decisions made

- cspell is enforced as a **single root gate**, not per-app — matches the plan's
  "Universal / every PR" intent and avoids re-fixing 11 inconsistent scripts.
- Generated/build trees are spell-ignored at the config level; legacy exercise dirs are
  left out of the spell scope (consistent with their pending standardization decision).
