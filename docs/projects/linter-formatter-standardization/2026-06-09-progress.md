# 2026-06-09 Progress

## Summary

Implemented Phase 3b: every app declares its checks as mise tasks, and every
app has per-app CI running them. The three apps with no CI (calendar-visualizer,
davidjfelix.com, ravrun) are covered for the first time.

## Work completed

- **Per-app `mise.toml`** in all five apps declaring `typecheck`, `lint`,
  `format`, `format:fix`, `build`, `test` (where a suite exists), and a `check`
  umbrella task. Tasks delegate to `package.json` scripts so pnpm-only usage
  still works. In Astro apps, `typecheck` has `wait_for = ["build"]` because
  `astro check` and `astro build` both regenerate `.astro/` types and clobber
  each other when the umbrella runs them in parallel (observed locally).
- **`typecheck` scripts added** to the three Astro apps (`astro check`).
  calendar-visualizer needed `@astrojs/check` added as a dev dependency.
- **`format` scripts are now check-mode** (`biome format .` + `prettier
  --check`), with `format:fix` as the write-mode counterpart — mirrors the
  existing `lint` / `lint:fix` convention.
- **Prettier scoped to explicit `**/*.{md,mdx}` globs** in app scripts. Running
  `prettier --write .` from an app directory does not pick up the root
  `.prettierignore` (gitignore semantics: `*` ignores directories, so
  `!**/*.md` cannot re-include nested files — the root ignore file only ever
  matched root-level markdown). The old app scripts would have rewritten
  Biome-owned TS/HTML files had anyone run them.
- **Formatted the md/mdx debt** in calendar-visualizer, djf.io (CHANGELOG), and
  f311x. **Excluded `apps/djf.io/src/content/`** via app-level
  `.prettierignore`: the blog posts are authored with semantic line breaks (one
  sentence per line) and `proseWrap: always` would destroy that.
- **ravrun typecheck fixed**: added `skipLibCheck: true` to its hand-rolled
  `tsconfig.json` (every other app already had it directly or via
  `astro/tsconfigs/strict`). Without it, duplicate `@tanstack/router-core`
  copies in node_modules fail the build with TS2717/TS2403.
- **calendar-visualizer had two unformatted files** (`.config/cspell.json`,
  `src/pages/index.astro`) that no CI ever caught — fixed with
  `biome check --write`.
- **CI**: new `ci_calendar_visualizer.yml`, `ci_davidjfelix_com.yml`,
  `ci_ravrun.yml`; `ci_djf_io.yml` gained typecheck + format jobs;
  `ci_f311x.yml` gained a format job. All jobs now invoke `mise run <task>` so
  CI and local use the same entry point. Workflow-level
  `MISE_TRUSTED_CONFIG_PATHS: ${{ github.workspace }}` trusts the nested app
  configs. Paths filters now also include `.oxlintrc.json` and
  `.prettierrc.json`. actionlint, ghalint, zizmor, and pinact all pass.

## Decisions made

- Change detection stays the existing per-app workflow paths-filter pattern —
  no new tooling (per the 2026-05-15 note: keep it simple).
- The plan's "we do not add Prettier" stance was already superseded in the repo
  (root `.prettierrc.json`, `prettier` in mise tools, format scripts): Prettier
  owns md/mdx, Biome owns everything else. Plan updated to match.
- Blog content is excluded from automated prose rewrapping (authored style wins
  over formatter config). Open to revisiting if David prefers wrapped prose.
