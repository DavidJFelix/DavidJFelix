## Decisions

- **Type-checker policy (David):** use each framework's native type-checker; tsgo
  is only for when the alternative would be bare `tsc`. He's early-adopting tsgo as
  a tsc replacement, not as a replacement for framework-aware checkers (vue-tsc,
  svelte-check, astro check), which will adopt the native TS 7 engine in their own
  time. Result by app: Astro → astro check, SvelteKit → svelte-check, Nuxt →
  nuxt typecheck (vue-tsc), TanStack Start/React → tsgo.
- Pinned `vue-tsc` to 3.3.4 rather than 3.3.5: pnpm enforces a 24h
  `minimumReleaseAge` cooldown and 3.3.5 was ~15h old, which would have forced a
  `minimumReleaseAgeExclude` block in `pnpm-workspace.yaml`. 3.3.4 (2026-06-08)
  keeps the workspace file clean and identical in shape to the siblings.
- Declined the `@parcel/watcher` native build (`allowBuilds: false`): it only
  backs `nuxt dev`'s file watcher, not build/typecheck/CI.
