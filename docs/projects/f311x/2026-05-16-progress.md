# 2026-05-16 — Project kickoff

Created project from PRD: Effect-native AI agent app on Cloudflare for f311x.com. App lives at `apps/f311x/`.

## Decisions

- Stack confirmed: TanStack Start + Cloudflare Agents SDK + Alchemy v2 + Effect-TS
- Vercel AI SDK scoped to `AIChatAgent.onChatMessage` only; TanStack AI everywhere else
- OpenRouter primary, Anthropic direct fallback
- Zod everywhere by default

## Contradictions flagged

- `docs/projects/new-domain-sites/plan.md` had f311x.com listed as SvelteKit. Updated to TanStack Start with a note pointing at this project.

## Scaffold landed

- `bun create tanstack@latest f311x --no-install` -> `apps/f311x/`. Removed the nested `.git` it dropped.
- Switched to pnpm per CLAUDE.md (Wrangler projects stay on pnpm until bun-with-Wrangler is confirmed). `bun` stays only for `scripts/ingest.ts`.
- `pnpm typecheck` passes. `pnpm build` produces a Worker bundle (server + client + worker environments).

## API substitutions vs the PRD

PRD package / symbol names that did not match what's actually shipped:

| PRD | Reality |
|---|---|
| `@ai-sdk/openrouter` | `@openrouter/ai-sdk-provider` (community, official `@ai-sdk/openrouter` does not exist on npm) |
| `@tanstack/ai@alpha` | `@tanstack/ai@^0.18.0` (no `alpha` dist-tag) |
| `agents/vite-plugin` | `agents/vite` (default export `agents`) |
| `agents/tsconfig` | exists, but importing it as `extends` works -- the package ships `agents.tsconfig.json` |
| `createServerFnTool(...)` | `toolDefinition({...}).server(execute)` -- isomorphic def + `.server()` / `.client()` adapters |
| `@cloudflare/ai-chat` | required as an explicit dep; optional peer of `agents` |
| `Schedule.exponentialJittered(...)` | `Schedule.jittered(Schedule.exponential(...))` |
| `Effect.unit` | `Effect.void` |
| Alchemy v2 npm tag | `alchemy@next` resolves to `2.0.0-beta.40` -- usable |
