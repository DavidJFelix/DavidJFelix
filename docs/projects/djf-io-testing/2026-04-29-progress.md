## Decisions

- **Tests co-located, named `xxx.e2e.test.ts`** next to the source under test. Unit-test counterpart will be `xxx.test.ts` when Vitest lands.
- **`_` prefix on test files inside `src/pages/`**: Astro's filesystem router treats every `.ts` in `src/pages/` as an endpoint and gives no exclude config. The leading underscore is Astro's built-in escape hatch (any file/dir starting with `_` is skipped). Outside `src/pages/` the prefix isn't needed.
- **Vitest deferred**: no shared/util code in djf.io worth unit-testing yet. Plan documents the trigger to add it later.
- **Single browser (Chromium)**: blog is server-rendered HTML; cross-browser coverage adds CI minutes for marginal gain on this app.
- **`retries: 2` in CI**: standard Playwright practice for flake mitigation; locally `0`.
