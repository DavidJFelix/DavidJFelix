# 2026-06-11 — Scaffold session

## Summary

Scaffolded `apps/forzamonica.com` from scratch: a headless Shopify storefront on TanStack
Start + Cloudflare Workers, styled with PandaCSS + Ark UI, wired against mock.shop.

## Work completed

- Full app scaffold matching repo conventions (standalone pnpm project, mise check tasks,
  tsgo/Biome/Oxlint/Vitest, per-app biome.json)
- Storefront integration: `storefrontQuery` fetch client, GraphQL documents + types,
  catalog/product/cart server functions, httpOnly cart cookie
- Routes: landing, `/products`, `/products/$handle` (variant select, quantity, add to
  cart), `/cart` (line edit/remove, subtotal, checkout link)
- Panda recipes: `button` recipe + `quantityField` slot recipe for Ark NumberInput — the
  repo's first Ark UI usage and the reference pattern for styling Ark components
- CI/CD workflows; deploy is `vite build` + `wrangler deploy` (wrangler-action) with
  per-app `FORZAMONICA_*` secrets
- Verified end to end in the workerd dev server: SSR renders mock.shop catalog; cart
  mutations (cartCreate etc.) confirmed working against mock.shop

## Decisions

- **Headless storefront**, not an embedded Shopify admin app
- **Plain wrangler deploy**, not Alchemy v2 (f311x is the Alchemy proving ground and is
  currently broken in prod)
- **Raw Ark UI + Panda** with `@pandacss/preset-panda` — no Park UI preset
- **mock.shop until the real store exists** — cutover is config-only
- Rendered plain-text `description` instead of `descriptionHtml` to avoid
  `dangerouslySetInnerHTML` (revisit if rich product copy is needed)
- Wrangler pinned at 4.100.0 (peer requirement of `@cloudflare/vite-plugin` 1.40.x),
  ahead of the 4.98.0 used elsewhere in the repo
