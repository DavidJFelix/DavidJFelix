## Decisions

- **Methodology**: tried-and-true progressive mileage (Higdon-shaped) + Riegel/Daniels-lite paces,
  not a bespoke model. Encoded as rules, not literal plan tables, so duration and schedule shape
  stay flexible.
- **Race date is the anchor**; duration is chosen (default 18wk marathon). The "how long do I
  actually need" solver arrives later as the feasibility engine inverted (`suggestedWeeks` seeds
  it).
- **Stepback every 4th week at 0.8×** (David's lived plan), not the every-3rd/75% first proposed.
- **Peak long run 20mi / 2-week marathon taper** as defaults (David's ICS runs 22mi / 1-week — a
  one-line constants change if he prefers).
- All constants centralized; changing one should consciously break an anchor test.
- Post-race continuation is out of scope for now; the phase model leaves room for appended blocks.
