## Decisions

- **Granularity**: per-export parts with real dependencies, not undirected components. An export
  that imports another export depends on that part (exact version) with specifiers rewritten in the
  emitted files; internals reached by several parts become shared `--internal-*` parts;
  mutually-cyclic entries merge into one part.
- **Barrels are skipped, not decomposed into parts**: a re-export-only entry (like root `mod.js`) is
  the thing being decomposed; consumers of a part import the part.
- **Republish target**: real scoped publish under `@pkgdog/*` on npm. The spike stops at
  `npm publish --dry-run` (all 50 pass); a real publish needs the npm org + token
  (`PKGDOG_PUBLISH=1 mise run publish-parts` from `apps/pkg.dog/spike/`).
- Parts reuse the upstream version verbatim for now; a real pipeline needs its own version epoch.

  middle is not.
