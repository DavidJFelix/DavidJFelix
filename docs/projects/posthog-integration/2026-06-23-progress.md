Implemented the djf.io pilot end to end on the feature branch. It stays dark until the PostHog

## Decisions

- **Region:** PostHog Cloud **US** (`us.i.posthog.com` ingestion, `us-assets.i.posthog.com` assets,
  `ui_host: https://us.posthog.com`). Immutable once the project is created.
- **Proxy topology:** **same-origin Astro endpoint at `/diag`**, served by the `@astrojs/cloudflare`
  adapter (one `prerender = false` route on an otherwise fully static site). Same-origin is the
  strongest blocker-dodge and needs no new DNS.
- **Privacy posture:** **cookieless, storage-free** (`persistence: 'memory'`,
  `person_profiles: 'identified_only'`, autocapture + session replay off, pageview + pageleave on).
  No consent banner required.
  - **Open question / trade-off:** on this multi-page site `memory` persistence means every page
    load is a fresh anonymous id (no returning-visitor or session stitching). If that matters more
    than zero client storage, switch `persistence` to `'localStorage'` (still cookieless).

## Adapter knock-ons (handled)

Adopting the Cloudflare adapter to host one endpoint pulled in several things, each resolved so the
static site is otherwise unchanged:

- **Prerender moved to workerd** â†’ OG-image route's native `sharp` failed. Fixed with
  `prerenderEnvironment: 'node'`.
- **Image optimization** â†’ the adapter hands images to Cloudflare Images and ships them unoptimized
  (banner 333kB â†’ 3.7MB). `imageService: 'custom'` keeps Astro's build-time sharp service.
- **Sessions** â†’ adapter auto-injects a KV `SESSION` binding (auto-provisioned on deploy). We don't
  use sessions, so an in-memory driver drops the binding.
- **Sentry** â†’ `@sentry/astro` requires `@sentry/cloudflare` once an adapter is present. Installed;
  server instrumentation stays off.
- **Vitest** â†’ the adapter's Vite plugin rejects `getViteConfig`'s `resolve.external`. The adapter
  is skipped when `process.env.VITEST` is set.
