## Session summary

David asked for a full critique of djf.io ("criticize everything"), then triaged the findings into a
concrete change list. Work is bucketed: Claude drives the engineering items solo; copy and
design-taste items are decided with David; a few are backlog or GitHub issues. This session landed
the first zero-risk batch (no decisions needed).

## Done this batch

- Removed the unused `houston.webp` Astro starter asset.
- Proofread all three posts: fixed typos and grammar, removed every em-dash (straight punctuation
  only), converted curly quotes to straight.
- Added `bin/check-prose.ts`, a guard that fails on em-dashes or curly quotes in blog content, wired
  into the `check` task as `lint:prose`.
- Rewrote the two AI-generated image alt texts to describe the image instead of dumping the
  generation prompt.
- Replaced the deprecated `navigator.platform` with `navigator.userAgentData?.platform` plus a
  `userAgent` fallback, keeping the no-flash inline script.
- Footer is now a plain "© David J Felix" signature; dropped "all rights reserved" (no license
  language on the site).

## Decisions

- No em-dashes anywhere, straight quotes only. Enforced by the new prose guard.
- No license language on the site. The footer is a signature; the repo LICENSE covers source only.
- Drop rel=me / h-card: it does nothing for Bluesky (ATProto DID plus the `.well-known` handle
  already cover identity). Profile linking will instead ride on `Person.sameAs` JSON-LD in the SEO
  work.
- Image git-size optimization deprioritized to backlog: Astro already optimizes the served image, and
  re-encoding does not purge git history.
- AI disclosure will be a small, consistent per-post footer note (not a separate "colophon"), framed
  as craft rather than a disclaimer.

## Done, later in the session

- Fixed two e2e regressions the first batch introduced (platform detection broke the Mac hotkey-hint
  test and had a real bug reaching Mac Chrome users; the alt-text rewrite broke the hero-image
  locator). Lesson applied: unit + e2e now run before every commit.
- Extracted `PostCard.astro` from the three diverging card copies (home, blog index, tag page). One
  link stretches over the card via a `::after` overlay so the whole card is a click target; tag
  chips sit above it with z-index and stay independently clickable; the tag page marks the current
  tag with `aria-current="page"` and its chips are real links again. Container-API unit tests plus
  e2e for card-body clicks and chip precedence.
- Replaced the overflowing single-row nav with a hamburger menu below `md` (David's pick): inline
  links collapse behind a toggle with `aria-expanded`/`aria-controls`, Escape closes, icons swap via
  CSS from the button state, and the Search keyboard hint only renders from `md` up. New mobile e2e
  suite pins a 390px viewport and guards against horizontal overflow.
- PR #311 now tracks the branch.

## Done, third batch

- Swapped the desktop nav's Twitter word for the X logo glyph; the accessible name and the mobile
  menu still say Twitter (David's call: use the logo, never the name).
- Font pass: preload the two latin woff2 files that style above-the-fold text. Skipped manual latin
  subsetting: fontsource@5 ships no per-subset CSS and its unicode-range rules already stop other
  subsets from downloading. Two removals were walked back in review: the static Inter package stays
  because the OG image generator resolves its woff files at build time, and JetBrains Mono stays as
  the mono face on David's request, now with system fallbacks behind it.
- Structured data: Person plus WebSite JSON-LD on home (Person on About) with `sameAs` profile
  links, BreadcrumbList on posts, and prev/next neighbor links on each post as crawlable internal
  linking. Helpers live in `src/lib/structured-data.ts` and `postNeighbors` in `src/lib/blog.ts`,
  both fully unit tested.
