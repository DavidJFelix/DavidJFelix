### docs(repo): add a code style guide

New `docs/contributing/code-style.md` documents how code is shaped, complementing the existing
which-language (scripting-style) and which-tool (tooling-standard) guides. It codifies: formatters
own formatting, with language norms first and house defaults (two-space, tight spacing, no
semicolons, single quotes, K&R braces) filling the gaps -- the JS/TS side of which was already
live in `biome.jsonc`; functional, data-centric shape (immutable data plus co-located free
functions; classes only when they earn it; pipelines over loops unless performance says
otherwise); named position-independent arguments past two params, with the
`XxxParams`/`XxxProps`/`XxxOptions` naming convention; TypeScript `interface`-over-`type` with the
`Prettify<T>` escape hatch, `import type`, named exports, and shoehorn-not-`as` in tests; Rust
clippy-pedantic-and-fix-everything with no `unwrap` outside tests; and a hard bash budget of
fewer than 10 trivial lines before a script must become Bun/TypeScript (or a purpose-built
Rust/Go utility for recurring workflows).

Alongside it: `testing.md` gains parameterized tests (`test.each` over copy-paste, replacing
`describe` grouping), given/when/then body order, and property-based/fuzz guidance (fast-check,
proptest); `scripting-style.md`'s bash section now carries the explicit 10-line budget; the guide
is indexed in `CONTRIBUTING.md` and summarized as a hard rule in `AGENTS.md`.

## 2026-07-01

### fix(ci): stop preview workflows duplicating their sticky PR comment

The preview-comment upsert only read the first page of a PR's comments
(`GET /issues/{n}/comments?per_page=100`). That endpoint returns comments oldest-first and has no
sort option, so once a PR crossed ~100 comments the sticky comment fell off page 1, the marker was
never found, and every rerun posted a fresh duplicate. `bin/comment-preview.ts` now walks every
page, collecting all marker-bearing comments; it edits the oldest in place and deletes the rest,
which both fixes forward and cleans up duplicates earlier runs already left on open PRs. Duplicate
DELETEs treat a 404 as a no-op so concurrent runs racing on the same stale comment don't abort the
cleanup.

Consolidated the two copies while here: `apps/f311x/bin/comment-preview.ts` was a near-identical
fork, so `cd-preview-f311x.yml` now calls the shared root script (via `$GITHUB_WORKSPACE`) with
`PREVIEW_MARKER` / `PREVIEW_TITLE` / `PREVIEW_FOOTER`, matching how the `preview-wrangler` composite
already drives it. The new `PREVIEW_FOOTER` input preserves f311x's Alchemy-stage teardown wording;
the wrangler apps keep the default version phrasing. One script, one place to fix, all 11 preview
workflows covered.

### fix(f311x): repair the wip github.ts CI-credentials stack

The wip Alchemy stack pinned Stack's generics as `<void, ConfigError>`, but Stack's second type
parameter is the provider requirements, not the error channel (ConfigError is already hardwired

cleanly, so it keeps real coverage from the tsgo gate.
