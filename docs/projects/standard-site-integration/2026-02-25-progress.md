# 2026-02-25 — Initial Implementation

## Summary

Built the core standard.site integration for djf.io, covering publication verification, document link tags, and a sync script.

## Work Completed

- Created `src/standard-site.ts` config module with DID (`did:plc:nlbldots3jn3lk6mzca4rqzm`) and AT-URI helpers
- Added `.well-known/site.standard.publication` Astro page endpoint
- Added named `head` slot to `BaseLayout.astro` for injecting link tags
- Added `<link rel="site.standard.document">` to `BlogPost.astro` (renders when `atUri` is present in frontmatter)
- Extended blog content schema with optional `atUri` field
- Created `scripts/standard-site-sync.ts` that:
  - Authenticates with Bluesky PDS
  - Creates/updates `site.standard.publication` record
  - Syncs each blog post as a `site.standard.document` record
  - Writes `atUri` back to post frontmatter
- Added `sync:standard-site` script to package.json

## Decisions

- Used `self` as the publication record rkey (convention for singleton records)
- Used blog post filenames as document rkeys for stable, deterministic IDs
- Used `node --experimental-strip-types` to run TypeScript sync script directly
- Kept the sync as a manual script (not part of build) to avoid requiring auth secrets at build time
