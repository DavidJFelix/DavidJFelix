## 2026-07-01

### fix(ci): stop preview workflows duplicating their sticky PR comment

The preview-comment upsert only read the first page of a PR's comments
(`GET /issues/{n}/comments?per_page=100`). That endpoint returns comments oldest-first and has no
sort option, so once a PR crossed ~100 comments the sticky comment fell off page 1, the marker was
never found, and every rerun posted a fresh duplicate. `bin/comment-preview.ts` now walks every
page, collecting all marker-bearing comments; it edits the oldest in place and deletes the rest,
which both fixes forward and cleans up duplicates earlier runs already left on open PRs. Duplicate
DELETEs treat a 404 as a no-op so concurrent runs racing on the same stale comment don't abort the
cleanup.

Consolidated the two copies while here: `apps/f311x/bin/comment-preview.ts` was a near-identical
fork, so `cd-preview-f311x.yml` now calls the shared root script (via `$GITHUB_WORKSPACE`) with
`PREVIEW_MARKER` / `PREVIEW_TITLE` / `PREVIEW_FOOTER`, matching how the `preview-wrangler` composite
already drives it. The new `PREVIEW_FOOTER` input preserves f311x's Alchemy-stage teardown wording;
the wrangler apps keep the default version phrasing. One script, one place to fix, all 11 preview
workflows covered.

### fix(f311x): bump wrangler to 4.105.0 to satisfy vite-plugin peer dependency

Lockfile maintenance advanced `@cloudflare/vite-plugin` to 1.42.3, whose `wrangler` peer dependency
is `^4.105.0`, but the app still pinned `wrangler` at 4.103.0 -- so `vite build` failed CI with "The
installed version of Wrangler does not satisfy the peer dependency required by
@cloudflare/vite-plugin (^4.105.0)". Pinned `wrangler` to 4.105.0 (the minimum satisfying version)
and regenerated the lockfile.

### docs(repo): consolidate agent entry points and add review personas

Agent access had sprawled: `CLAUDE.md` carried ~180 lines duplicating `AGENTS.md`,
`CONTRIBUTING.md`, and the loose style guides at the `docs/` root, and the copies had already
drifted (CONTRIBUTING still described the retired underscore workflow naming). The path is now
`CLAUDE.md` (thin shim importing `AGENTS.md`) -> `AGENTS.md` (canonical entry: key paths, hard
rules, doc map) -> `CONTRIBUTING.md` (PR workflow + guide index) -> `docs/contributing/` (the mini
guides). Moved `configuration-style.md`, `scripting-style.md`, `github-actions-style.md`, and
`tooling-standard.md` into `docs/contributing/` via `git mv`; extracted `file-naming.md`,
`testing.md`, `project-docs.md`, and `linting.md` from the old `CLAUDE.md`/`CONTRIBUTING.md`
prose; folded the ecosystem tool choices and the `mise.lock` rule into `tooling-standard.md`.
`docs/agents/issue-tracker.md` is now the canonical home of the human-task issue format.

Added six review personas as `.claude/agents/` subagents -- `design-reviewer`,
`product-reviewer`, `engineering-reviewer`, `testing-reviewer`, `benchmark-reviewer`,
`tooling-reviewer` -- each a one-question rubric with explicit "not yours" boundaries and a capped
report ("just enough" applies to reviews too). A new `/panel-review` skill pins the diff, seats
the relevant personas, fans them out in parallel, and reports verdicts side by side without
re-ranking them; `/review` keeps standards/spec conformance.

### chore(repo): make .agents/ the solid home for all skills and review personas

The vendored mattpocock skills already followed the pattern -- solid copies in `.agents/skills/`,
tracked symlinks in `.claude/skills/` -- but the repo-authored surfaces did not. Moved
`dependency-freshness`, `panel-review`, and the six review personas into `.agents/` and left
per-item symlinks behind (`.claude/skills/<name>` -> skill folder, `.claude/agents/<name>.md` ->
agent file), so `.agents/` is now the tool-agnostic source of truth and `.claude/` is purely the
Claude Code discovery surface. Per-item symlinks are the documented-supported form and were
verified in-session (skills hot-register and personas spawn through them); whole-directory
symlinks have known upstream regressions and are avoided. No configuration alternative exists --
Claude Code has no setting to relocate skill/agent discovery, and the plugin route would impose
`plugin:name` namespacing. Conventions recorded in `AGENTS.md`.
