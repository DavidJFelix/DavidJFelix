# Dependency Freshness

## Goal

Keep every package in the repo current. Nothing should fall significantly behind.

## Scope

- mise as the source of truth for tool versions
- Audit current package and tool versions across the monorepo
- Decide between monorepo-wide settings vs. per-app config
- Author a reusable skill / workflow for version-bump cycles
- Establish a cadence for sweeps
