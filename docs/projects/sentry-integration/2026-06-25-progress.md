## Decisions

- **Refined relay everywhere** (per the user): every app gets the `/bugs` tunnel, not direct
  `*.ingest.sentry.io` POSTs. The static Astro sites and the ravrun SPA gained a server (Cloudflare
  adapter / a small worker) to host it.
- **Client SDK per framework**: `@sentry/astro` (3 Astro sites), `@sentry/react` (4 TanStack Start +
  ravrun), `@sentry/browser` (SvelteKit, Nuxt).
- **Server-side capture, tiered**: the four TanStack Start apps and ravrun wrap their fetch handler
  with `@sentry/cloudflare`'s `withSentry` (workerd-native, no `@sentry/node`), reading the same
  `VITE_PUBLIC_SENTRY_DSN` that Vite inlines into the client build â€” **no runtime worker var** (see
  "DSN source" below). The 3 Astro sites stay client-only (prerendered; djf.io precedent).
  **Deferred** on SvelteKit (monicandavid) and Nuxt (pkg.dog): their server SDKs are
  `@sentry/node`-based and won't run on workerd, and both are low-traffic placeholders â€” the tunnel
  already captures client errors. Revisit with `@sentry/cloudflare` if SSR visibility is needed.
- **Gating invariant**: the browser SDK is gated on the inlined build-time `*_SENTRY_DSN` literal
  (Astro `PUBLIC_`, Vite `VITE_PUBLIC_`) so Vite/Astro dead-code-eliminates it in keyless builds â€”
  verified each keyless client bundle ships **zero** Sentry code. pkg.dog gates on Nuxt
  `runtimeConfig` (`NUXT_PUBLIC_*`), baked at build from the same env (`nuxt.config.ts` reads
  `process.env`), so its SDK is a lazy chunk loaded only when the value is set.

## DSN source: one build var, no runtime worker var

Server-side `withSentry` reads the **build-inlined** `VITE_PUBLIC_SENTRY_DSN` (the value Vite
already bakes into the client), not a runtime worker var â€” so activation matches djf.io exactly: set
one repo variable, push, done. Nothing is set in Cloudflare.

- **forzamonica / revision.city / startchi**: worker is Vite-built (`@cloudflare/vite-plugin`), so
  `import.meta.env.VITE_PUBLIC_SENTRY_DSN` inlines into `src/server.ts`.
- **f311x**: same, via Alchemy's `Cloudflare.Vite` build.
- **ravrun**: adopted `@cloudflare/vite-plugin` for this â€” its worker was previously
  wrangler-bundled, where `import.meta.env` wouldn't inline.
- **pkg.dog**: `nuxt.config.ts` bakes `runtimeConfig.public` from the `NUXT_PUBLIC_*` build env.
