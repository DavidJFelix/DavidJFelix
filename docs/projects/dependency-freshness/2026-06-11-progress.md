# 2026-06-11 Progress

## Summary

Portfolio review session — planning only. Phase 5 (gated auto-merge) was
unblocked by Linter/Formatter Phase 3b landing 2026-06-09, but David lowered
its priority: PR volume is currently manageable, so unattended merging is no
longer urgent. At the same time the bar for it went up.

## Decisions made

- **Priority lowered.** Phase 5 is parked; it is not the next thing.
- **Bar raised.** No auto-merge without a real test suite (`passWithNoTests`
  doesn't count). Web apps additionally need per-PR preview deployments with
  smoke and screenshot tests — see the new
  [preview-deployments](../preview-deployments/plan.md) project, which this
  phase now depends on. Motivating example: f311x shipped broken on green CI.
- **Mechanism settled.** GitHub branch-protection required checks don't fit
  the paths-filtered per-app CI (untouched apps never report a status). Gate
  per-PR via Renovate `automerge` / GitHub auto-merge instead.
- **Renovate owns everything** — npm, mise, and Cargo manifests plus
  `lockFileMaintenance` for transitives. The skill no longer owns manifest
  bumps; its residual role (or retirement) is an open question.

## Next steps

  `lockFileMaintenance` on) — small async task, not gated on anything.
- Confirm Dependabot is off where Renovate is on.
- Revisit auto-merge once preview-deployments lands and apps have real suites.

## Blockers or open questions
