# 2026-05-29 Progress

## Summary

Reopened scope during a project recap. The pipeline produces update PRs but still
needs a human to merge them, so it isn't actually unattended. Added a full-automation
phase to the plan.

## Decisions

- The project is **not done**. Gated auto-merge is required before it can be
  considered complete.
- Every package in an update PR must run the full check set for its subtree —
  **typecheck, lint, format, and tests** — so results can automatically gate the PR.
- **Green → auto-merge** (Renovate `automerge` + matching auto-merge for
  skill-produced PRs). **Red → hold + `deps:needs-attention`.**
- Majors and known-incompatible bumps **never** auto-merge regardless of checks.
- Auto-merge only enables per-package once that package declares its subtree
  checks — hard dependency on Linter/Formatter Standardization Phase 3b.

- For multi-package batch PRs, do we require the union of all affected subtree
  checks to pass before auto-merge, or gate per-package? (Leaning: union must pass.)
