# 2026-06-25 — Full-fleet rollout

Wired PostHog into the remaining ten apps on branch `claude/pensive-euler-w2a96w`, templatizing the
djf.io pilot. One commit per app; everything ships **dark** (no analytics until a project key is
provided). Runs back-to-back with the Sentry rollout (shared relay scaffolding, same gating).

## Decisions

- **Same-origin reverse proxy everywhere** (`/diag`, PostHog US cloud): events and the lazily-loaded
  SDK ride this first-party path, dodging blockers that drop `*.posthog.com`. The static Astro sites
  and ravrun gained a server to host it.
- **Cookieless / storage-free** init everywhere (`persistence: 'memory'`, `identified_only`,
  autocapture + replay off, pageview + pageleave on) — the djf.io posture; no consent banner.
  (Trade-off unchanged: `memory` means a fresh anon id per load; switch to `localStorage` later if
  returning-visitor stitching matters.)
- **Proxy host per stack**: an Astro on-demand route (`src/pages/diag/[...path].ts`, 3 sites),
  TanStack Start server routes (`src/routes/diag/$.ts`, 4 apps), a SvelteKit endpoint
  (`src/routes/diag/[...path]/+server.ts`, monicandavid), a Nitro route
  (`server/routes/diag/[...].ts`, pkg.dog), and a hand-written worker (`src/worker.ts`, ravrun). All
  reuse the pure, 100%-covered `postHogUpstream` host-router.
- **Gating**: `posthog-js` loads only when the key is set — gated on the inlined build literal
  (`PUBLIC_POSTHOG_KEY` / `VITE_PUBLIC_POSTHOG_KEY`) so keyless builds DCE it (verified zero PostHog
  code in each keyless client bundle). pkg.dog uses Nuxt `runtimeConfig` (`NUXT_PUBLIC_POSTHOG_KEY`,
  a runtime worker var), so its SDK is a lazy chunk loaded only when set.

## Verification

Per app: typecheck, lint, format, unit (the `posthog-proxy` host-router is 100%-covered), build, and
a `wrangler dev` / `vite preview` smoke all green. The proxy's live upstream is only reachable from
a real Cloudflare deploy, so the proxy↔PostHog round-trip is confirmed at preview/prod, not local
boots (per the djf.io e2e convention).
